## リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| レイアウト変更による既存機能の破損 | 高 | 段階的リリース、フィーチャーフラグ |
| モバイル対応の工数増 | 中 | デスクトップ優先、モバイルは最小限 |
| パフォーマンス低下（2パネル同時描画） | 中 | React.memo、useMemoの活用 |
