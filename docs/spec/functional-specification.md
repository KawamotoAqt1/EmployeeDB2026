## 1. システム概要

### 1.1 目的

SharePointリストベースの社員データベースをWebアプリケーションとして再構築し、スキル管理をタグベースに改善することで検索性と管理性を向上させる。

### 1.2 対象ユーザー

| ユーザー | 用途 |
|----------|------|
| 営業部門 | スキルマッチングによる人材検索、案件アサイン管理 |
| 人事部門 | 社員情報の登録・更新・管理 |
| マネージャー | 稼働状況の把握、チーム構成の確認 |
| システム管理者 | ユーザー管理、マスタデータ管理 |

### 1.3 成功基準

- スキル検索精度 100%
- 入力工数 50% 削減
- 対象社員を1分以内に特定可能

## 3. 認証・認可

### 3.1 認証方式

- JWT (JSON Web Token) ベースのステートレス認証
- トークン有効期限: 7日間（設定変更可能）
- パスワード: bcrypt ハッシュ化（ソルトラウンド: 10）
- トークン格納: ブラウザの localStorage

### 3.2 ロール定義

| ロール | 説明 | 権限範囲 |
|--------|------|----------|
| **ADMIN** | システム管理者 | 全機能の利用可能（ユーザー管理含む） |
| **EDITOR** | 編集者 | 社員・タグ・企業・案件の登録・編集・削除が可能 |
| **VIEWER** | 閲覧者 | 全データの閲覧のみ可能 |

### 3.3 権限マトリクス

| 機能 | ADMIN | EDITOR | VIEWER |
|------|:-----:|:------:|:------:|
| 社員一覧・詳細閲覧 | o | o | o |
| 社員登録・編集・削除 | o | o | x |
| 写真アップロード | o | o | x |
| タグ管理 | o | o | x |
| 企業一覧・詳細閲覧 | o | o | o |
| 企業登録・編集・削除 | o | o | x |
| 案件一覧・詳細閲覧 | o | o | o |
| 案件登録・編集・削除 | o | o | x |
| 参画アサイン管理 | o | o | x |
| ユーザー管理 | o | x | x |
| CSVインポート | o | x | x |
| パスワード変更（自分） | o | o | o |

### 3.4 認証フロー

```
1. ユーザーが /login でメール・パスワードを入力
2. POST /api/auth/login → JWTトークンを返却
3. トークンをlocalStorageに保存
4. 以降のAPIリクエストに Authorization: Bearer {token} ヘッダーを付与
5. バックエンドでトークン検証 → req.user にユーザー情報を格納
6. 401レスポンス受信時はトークンをクリアし /login にリダイレクト
```

## 5. 社員管理機能

### 5.1 社員一覧画面

#### レイアウト

- **デスクトップ (lg 以上)**: マスター・ディテール形式（左35% 一覧 / 右65% 詳細）
  - 一覧から社員を選択すると右ペインに詳細を表示
  - 初期表示時は最初の社員を自動選択
- **タブレット・モバイル**: カードグリッドレイアウト
  - カードクリックで詳細画面へ遷移

#### ページネーション

- 1ページあたり12件表示
- 前へ/次へボタン、ページ番号表示
- 総件数・総ページ数表示

#### フィルター・検索

| フィルター | 種類 | 動作 |
|-----------|------|------|
| キーワード検索 | テキスト入力 | 300ms デバウンスによる自動検索 |
| 検索モード | トグルボタン | 部分一致 / 先頭一致の切り替え |
| ステータス | ドロップダウン | 在籍/休職/退職(求職)/退職 |
| 部署 | ドロップダウン | 部署名でフィルタ |
| 役職 | ドロップダウン | 役職名でフィルタ |
| スキルタグ | タグピッカー | カテゴリ別タグ選択（AND条件） |

#### キーワード検索の対象フィールド

- 氏名、氏名カナ、社員番号、メールアドレス
- 部署、役職、勤務地、国
- 住所、最寄り駅、備考

#### スキルタグ検索

- カテゴリタブでカテゴリ切り替え
- タグ名で絞り込み検索
- 複数タグ選択可能（チェックボックス）
- AND条件: 選択した全タグを持つ社員のみ表示
- OR条件: 選択したタグのいずれかを持つ社員を表示
- 選択中のタグをバッジで表示

#### フィルター操作

- フィルター変更時に選択中の社員をリセット
- リセットボタンで全フィルターをクリア
- 詳細フィルターの展開/折りたたみ
- アクティブフィルター数のインジケーター表示

### 5.2 社員詳細画面

#### 表示セクション

1. **プロフィールカード**: 写真、氏名、社員番号、ステータスバッジ
2. **基本情報グリッド**: 部署、役職、入社日、契約形態、勤務地 等
3. **連絡先・住所情報**: メール、国、住所、最寄り駅
4. **備考セクション**: テキスト表示
5. **スキルセクション**: カテゴリ別にグループ化、レベル表示（クリックで編集可能、EDITOR以上）
6. **参画履歴セクション**: 案件アサイン一覧（新しい順）

#### 操作（EDITOR以上）

- 編集ボタン → 編集フォームへ遷移
- 削除ボタン → 確認ダイアログ → 削除実行
- 写真アップロード/変更

### 5.3 社員登録・編集フォーム

#### フォームセクション

1. **基本情報**
   - 社員番号（必須、1〜20文字）
   - 氏名（必須、1〜100文字）
   - 氏名カナ
   - メールアドレス（メール形式バリデーション）
   - 生年月日（日付入力）
   - 性別（男性/女性/その他）
   - 契約形態（正社員/契約社員/パートタイム/派遣/インターン/BP）

2. **組織情報**
   - 部署（ドロップダウン）
   - 役職（ドロップダウン）
   - 勤務地（ドロップダウン）
   - ステータス（在籍/休職/退職(求職)/退職）
   - 入社日
   - 契約終了日

3. **住所・連絡先**
   - 国
   - 最寄り駅
   - 住所（テキストエリア）

4. **備考**（テキストエリア）

5. **スキル**
   - SkillTagSelector コンポーネントでスキル追加
   - カテゴリ選択 → タグ選択 → レベル選択 → 追加
   - 登録済みスキルの表示と削除

#### バリデーション

- 社員番号: 必須、1〜20文字、ユニーク
- 氏名: 必須、1〜100文字
- メール: メール形式（任意）
- 日付: ISO 8601形式

### 5.4 写真アップロード

- 対応形式: JPEG, PNG, GIF, WebP
- 最大サイズ: 5MB
- 保存先: `/uploads/employee-{timestamp}-{random}.{ext}`
- 更新時は旧ファイルを自動削除
- multipart/form-data で送信

## 7. 企業管理機能

### 7.1 企業一覧画面

- テーブル形式で表示（企業名、業種、ステータス、登録日）
- キーワード検索（企業名）
- フィルター: 業種、ステータス（ACTIVE/INACTIVE/TERMINATED）
- ページネーション
- ステータスバッジ表示

### 7.2 企業詳細画面

#### 表示セクション

1. **企業情報カード**: 企業名、業種、Webサイト、備考
2. **拠点一覧**: 拠点名、住所、電話番号、本社フラグ
3. **部署階層ツリー**:
   - ツリービューで表示（インデント付き）
   - 展開/折りたたみ対応
   - 子部署追加ボタン
   - 編集・削除ボタン（子部署がない場合のみ削除可能）
   - 循環参照防止チェック
4. **担当窓口一覧**: 担当者名、役職、メール、電話、携帯
5. **関連案件一覧**: その企業に紐付く案件

#### 部署操作モーダル

- 部署タイプ: 事業部/部/課/ユニット/その他
- 親部署選択（ドロップダウン）
- 拠点選択（任意）

### 7.3 企業登録・編集フォーム

#### フォームセクション

1. **基本情報**: 企業コード（ユニーク）、企業名、フリガナ、業種、ステータス、Webサイト、郵便番号、電話番号、住所
2. **拠点**: 動的追加/削除（拠点名、住所、本社フラグ）
3. **担当窓口**: 動的追加/削除（担当者名、役職、メール、電話、携帯、主担当フラグ）
4. **備考**

- 企業作成時に拠点・担当窓口をまとめてトランザクションで登録

## 8. 案件管理機能

### 8.1 案件一覧画面

- カードグリッド形式（レスポンシブ: 1〜3列）
- 各カードに企業名、契約期間、アサイン人数を表示
- フィルター: キーワード、ステータス、契約形態
- ページネーション
- ステータスバッジ表示

### 8.2 案件ステータス定義

| ステータス | 英語名 | 説明 |
|-----------|--------|------|
| 提案中 | PROPOSAL | 提案・見積もり段階 |
| 進行中 | IN_PROGRESS | 契約締結済み、作業中 |
| 保留 | ON_HOLD | 一時停止中 |
| 完了 | COMPLETED | 作業完了 |
| 中止 | CANCELLED | キャンセル済み |

### 8.3 契約形態定義

| 形態 | 英語名 | 説明 |
|------|--------|------|
| 派遣 | DISPATCH | 派遣契約 |
| SES | SES | 準委任契約 |
| 請負 | CONTRACT | 請負契約 |

### 8.4 案件詳細画面

#### 表示セクション

1. **案件情報カード**: 案件名、説明、企業名、部署、契約情報（形態、期間、納期）
2. **予算情報**: 予算額、単価
3. **参画アサイン一覧**:
   - 社員名、役割、期間、稼働率（%）、ステータス
   - 追加/編集/削除ボタン（EDITOR以上）

#### アサイン操作モーダル

| フィールド | 説明 |
|-----------|------|
| 社員 | ドロップダウン（全社員から選択） |
| 役割 | テキスト入力 |
| 開始日 | 日付入力（必須） |
| 終了日 | 日付入力（開始日以降） |
| 稼働率 | 0〜100% |
| 単価 | 金額入力 |
| 請求タイプ | 時間/日/月/固定 |
| ステータス | 予定/進行中/完了 |
| 備考 | テキスト入力 |

### 8.5 案件登録・編集フォーム

#### フォームセクション

1. **基本情報**: 案件コード（ユニーク）、案件名、説明、ステータス、契約形態
2. **企業・部署**: 企業選択（ドロップダウン）→ 部署選択（企業に応じた動的フィルタ）
3. **契約期間**: 開始日、終了日、納品日
4. **予算・単価**: 予算（Decimal 15,2）、単価（Decimal 15,2）
5. **勤務地**
6. **備考**

#### バリデーション

- 案件コード: 必須、1〜50文字、ユニーク
- 案件名: 必須、1〜200文字
- 企業: 必須（有効なUUID）
- 日付: 終了日 >= 開始日

## 10. CSVインポート機能

### 10.1 概要

- ADMIN権限のみ利用可能
- CSVファイルの社員データを一括インポート
- プレビュー機能でインポート前にデータ確認

### 10.2 対応フォーマット

- 文字コード: UTF-8（BOM付き推奨）
- 区切り文字: カンマ（,）
- ヘッダー行: 必須

### 10.3 カラムマッピング

| CSV見出し | データベースフィールド |
|-----------|---------------------|
| 社員番号 | employeeNumber |
| 社員固有番号 | employeeUniqueNumber |
| 氏名 | fullName |
| 氏名（カナ） | fullNameKana |
| メールアドレス | email |
| 生年月日 | birthDate |
| 性別 | gender |
| 契約形態 | contractType |
| 部署 | department |
| 役職 | position |
| 勤務地 | location |
| 国 | country |
| 居住地 | residence |
| 最寄り駅 | station |
| 入社日 | hireDate |
| 契約終了日 | contractEndDate |
| ステータス | status |
| 備考 | remark |
| 写真URL | photoUrl |

### 10.4 Enum値の自動変換

| フィールド | 日本語入力 | 変換後の値 |
|-----------|-----------|-----------|
| 性別 | 男性, 男, MALE, M | MALE |
| 性別 | 女性, 女, FEMALE, F | FEMALE |
| 性別 | その他, OTHER | OTHER |
| 契約形態 | 正社員, FULL_TIME | FULL_TIME |
| 契約形態 | 契約社員, CONTRACT | CONTRACT |
| ステータス | 在籍中, ACTIVE | ACTIVE |
| ステータス | 休職中, INACTIVE | INACTIVE |

### 10.5 インポートフロー

```
1. POST /api/import/preview でプレビュー取得
2. ユーザーがデータ確認
3. POST /api/import/employees で本インポート実行
4. 行単位でトランザクション処理
5. updateExisting: true の場合、社員番号が一致する既存レコードを更新
```

## 11. 共通仕様

### 11.1 レスポンシブデザイン

| 画面幅 | 対応 |
|--------|------|
| デスクトップ (lg: 1024px以上) | マスター・ディテールレイアウト |
| タブレット (md: 768px〜1023px) | カードグリッド（2列） |
| モバイル (sm: 640px以下) | カードグリッド（1列） |

### 11.2 ページネーション

- デフォルト表示件数: 20件（社員一覧は12件）
- 最大表示件数: 100件
- ページ番号ナビゲーション + 前へ/次へ
- 総件数・総ページ数表示

### 11.3 ソート

- デフォルト: 社員番号昇順
- ソートフィールドとソート方向（asc/desc）を指定可能

### 11.4 データキャッシュ

- React Query によるクライアントキャッシュ
- キャッシュ有効期間: 5分（staleTime）
- ミューテーション成功時にキャッシュを自動無効化
- `keepPreviousData` による画面遷移時のちらつき防止

### 11.5 エラーハンドリング

#### APIレスポンス形式

**成功レスポンス:**
```json
{
  "success": true,
  "data": { ... },
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 150,
    "totalPages": 8
  }
}
```

**エラーレスポンス:**
```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "エラーメッセージ",
    "details": [ ... ]
  }
}
```

#### エラーコード一覧

| コード | HTTPステータス | 説明 |
|--------|---------------|------|
| UNAUTHORIZED | 401 | 認証が必要 |
| TOKEN_EXPIRED | 401 | トークン期限切れ |
| INVALID_TOKEN | 401 | 不正なトークン |
| USER_NOT_FOUND | 401 | ユーザーが存在しない |
| FORBIDDEN | 403 | 権限不足 |
| NOT_FOUND | 404 | リソースが存在しない |
| DUPLICATE_ENTRY | 409 | ユニーク制約違反 |
| DUPLICATE_CODE | 400 | コード重複 |
| DUPLICATE_EMAIL | 400 | メール重複 |
| VALIDATION_ERROR | 400 | バリデーションエラー |
| BAD_REQUEST | 400 | 不正なリクエスト |
| FOREIGN_KEY_ERROR | 400 | 参照先が存在しない |
| DATABASE_ERROR | 500 | データベースエラー |
| SERVER_ERROR | 500 | サーバー内部エラー |

### 11.6 確認ダイアログ

- 削除操作時には必ず確認ダイアログを表示
- 「削除してよろしいですか？」のメッセージ
- 確認ボタン（赤色）とキャンセルボタン

### 11.7 ローディング状態

- API通信中はボタンにローディング表示
- 二重送信防止（ローディング中はボタン無効化）
- ページ全体のローディングスピナー
