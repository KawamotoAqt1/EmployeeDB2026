# 社員データベースWebアプリ 企画会議

**開催日**: 2026年2月2日
**参加者**: プロジェクトオーナー、開発チーム
**文書バージョン**: 1.0

---

## 1. プロジェクト背景

### 1.1 現状の課題

現在、社員情報はSharePointリスト（employees01）で管理されているが、以下の課題がある：

| 課題 | 詳細 |
|------|------|
| **スキル検索の困難** | Skills, Assignedが自由記述のため、データにばらつきがあり検索精度が低い |
| **データの一貫性** | 入力者によって表記揺れが発生（例: Python / python / パイソン） |
| **検索性の限界** | SharePointの検索機能では複合条件での絞り込みが困難 |
| **拡張性** | 案件・企業情報との連携ができていない |

### 1.2 プロジェクトの目的

1. スキル情報をタグベースに改善し、**検索精度を向上**させる
2. 階層型タグシステムにより、**データの一貫性を確保**する
3. 将来的な案件・企業DBとの連携を見据えた**拡張性のある設計**
4. 権限管理により**データの安全性を確保**する

## 3. 機能要件

### 3.1 Phase 1: 社員DB + タグシステム（優先実装）

#### 社員管理
- 社員の一覧表示・詳細表示
- 社員情報の登録・編集・削除（管理者のみ）
- 社員写真の管理

#### タグシステム
- **階層型タグ構造**
  - カテゴリ > サブカテゴリ > タグ
  - 例: TECH > 言語 > Python
- **スキルレベル**
  - 各スキルに対して4段階のレベルを設定可能
  - 未経験 / 微経験(1〜3年) / 中堅(3〜5年) / ベテラン(5年以上)
- **タグマスタ管理**
  - タグの追加・編集・削除（管理者のみ）

#### 検索・フィルタ
- 氏名（部分一致）検索
- 基本属性フィルタ（部署、役職、勤務地、ステータス）
- スキルタグ検索（複数タグのAND/OR指定）
- スキルレベルでの絞り込み

#### 認証・権限
- ログイン/ログアウト
- 2ロール制
  - **管理者**: 全操作可能
  - **一般**: 閲覧のみ

#### データ移行
- SharePointからのCSVインポート機能

### 3.2 Phase 2: 案件・企業DB（将来実装）

- 企業マスタ管理
- 案件情報管理
- 社員⇔案件のアサイン管理
- アサイン履歴の追跡

## 6. 成功基準

| 指標 | 目標 |
|------|------|
| スキル検索精度 | 100%（タグ完全一致） |
| データ入力工数 | 現状比50%削減（タグ選択式により） |
| 検索所要時間 | 1分以内で目的の社員を特定可能 |

---

## 7. リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 既存データの移行エラー | 高 | インポート前のバリデーション、プレビュー機能 |
| タグ体系の不足 | 中 | 管理者によるタグ追加機能を実装 |
| ユーザー教育 | 中 | 操作マニュアル作成 |

---

## 8. 決定事項

1. **Phase 1として社員DB + タグシステムを優先実装する**
2. **スキルは階層型タグ + レベル制で管理する**
3. **認証は管理者/一般の2ロール制とする**
4. **ローカル開発環境で構築し、動作確認後に本番展開を検討する**
