## 1. 技術スタック

### 1.1 採用技術

| レイヤー | 技術 | 選定理由 |
|---------|------|---------|
| **フロントエンド** | React 18 + TypeScript | 型安全性、コンポーネント再利用、エコシステムの充実 |
| **スタイリング** | Tailwind CSS | 高速なUI開発、一貫したデザイン |
| **状態管理** | TanStack Query (React Query) | サーバー状態管理、キャッシュ、自動再取得 |
| **ルーティング** | React Router v6 | SPA標準のルーティング |
| **バックエンド** | Node.js + Express + TypeScript | フロントと言語統一、型共有可能 |
| **ORM** | Prisma | 型安全なDB操作、マイグレーション管理 |
| **データベース** | PostgreSQL 15 | ENUM対応、JSON対応、拡張性 |
| **認証** | JWT + bcrypt | ステートレス認証、パスワードハッシュ |
| **コンテナ** | Docker Compose | 開発環境の統一 |

### 3.2 テーブル詳細

#### employees（社員）
```sql
CREATE TABLE employees (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  employee_number VARCHAR(50) UNIQUE NOT NULL,
  employee_unique_number VARCHAR(50) UNIQUE,
  full_name VARCHAR(100) NOT NULL,
  full_name_kana VARCHAR(100),
  email VARCHAR(255) UNIQUE,
  birth_date DATE,
  gender VARCHAR(20) CHECK (gender IN ('男性', '女性', '設定なし')),
  contract_type VARCHAR(20) CHECK (contract_type IN ('正社員', '契約社員', 'BP')),
  department VARCHAR(50),
  position VARCHAR(50),
  location VARCHAR(50),
  country VARCHAR(50),
  residence TEXT,
  station VARCHAR(100),
  hire_date DATE,
  contract_end_date DATE,
  status VARCHAR(20) CHECK (status IN ('在籍', '休職', '退職（求職）', '退職')),
  remark TEXT,
  photo_url VARCHAR(500),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### tag_categories（タグカテゴリ）
```sql
CREATE TABLE tag_categories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  code VARCHAR(50) NOT NULL,
  name VARCHAR(100) NOT NULL,
  parent_id UUID REFERENCES tag_categories(id),
  sort_order INT DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### tags（タグ）
```sql
CREATE TABLE tags (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  category_id UUID REFERENCES tag_categories(id) ON DELETE CASCADE,
  name VARCHAR(100) NOT NULL,
  sort_order INT DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(category_id, name)
);
```

#### employee_skills（社員スキル）
```sql
CREATE TABLE employee_skills (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  employee_id UUID REFERENCES employees(id) ON DELETE CASCADE,
  tag_id UUID REFERENCES tags(id) ON DELETE CASCADE,
  level VARCHAR(30) CHECK (level IN ('未経験', '微経験(1～3年未満)', '中堅(3～5年)', 'ベテラン(5年以上)')),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(employee_id, tag_id)
);
```

#### users（ユーザー）
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  role VARCHAR(20) CHECK (role IN ('admin', 'viewer')) DEFAULT 'viewer',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 6. セキュリティ設計

### 6.1 認証フロー

```
1. ログイン
   Client → POST /api/auth/login { email, password }
   Server → { token: "JWT..." }
   Client → localStorage.setItem('token', token)

2. API呼び出し
   Client → GET /api/employees
            Headers: { Authorization: "Bearer {token}" }
   Server → JWT検証 → ユーザー情報取得 → レスポンス

3. ログアウト
   Client → localStorage.removeItem('token')
```

### 6.2 権限制御

```typescript
// Middleware
const requireAuth = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) return res.status(401).json({ error: 'Unauthorized' });
  // JWT検証...
  next();
};

const requireAdmin = (req, res, next) => {
  if (req.user.role !== 'admin') {
    return res.status(403).json({ error: 'Forbidden' });
  }
  next();
};
```

### 9.5 Nginx設定

```nginx
# /etc/nginx/sites-available/employee-db

server {
    listen 80;
    server_name employee-db.example.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name employee-db.example.com;

    # SSL証明書（Let's Encrypt）
    ssl_certificate /etc/letsencrypt/live/employee-db.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/employee-db.example.com/privkey.pem;

    # SSL設定
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers off;

    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;

    # フロントエンド（静的ファイル）
    location / {
        root /var/www/employee-db/frontend/dist;
        try_files $uri $uri/ /index.html;

        # キャッシュ設定
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }

    # API（バックエンド）
    location /api {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
```

## 11. 決定事項

1. **Prisma**をORMとして採用（型安全性重視）
2. **JWTトークン**はlocalStorageに保存（有効期限24時間）
3. **タグの階層**は2階層まで（カテゴリ > タグ）
4. **削除**は物理削除（論理削除は後日検討）
5. **写真**はURL参照のみ（ファイルアップロードは後日）
6. **本番環境**はさくらVPS + Nginx + PM2構成
7. **SSL**はLet's Encryptで無料証明書を使用
8. **バックアップ**は日次で7日間保持
