## 議事内容

### 1. 現状UIの確認と課題共有

**山田（PM）**: それでは会議を始めます。今回は現行の社員データベースUIを、オリジナルのSharePoint版デザインに近づける形で刷新したいという要望です。まず現状の課題を共有しましょう。

**田中（UX）**: 現行システムをユーザーに使ってもらったところ、いくつかフィードバックがありました。
- 社員一覧から詳細への遷移が「ページ遷移」なので、比較検討しにくい
- 検索結果を見ながら詳細を確認したいニーズが高い
- 写真がもっと大きく見たい、という声も

**佐藤（UI）**: オリジナルUIを見ると、**マスターディテール型のレイアウト**ですね。左に一覧、右に詳細。これはとても合理的です。現行のカード型一覧＋詳細ページ遷移より、情報の一覧性が高い。

**鈴木（FE）**: 技術的には、React Routerでのページ遷移を、サイドバイサイドのレイアウトに変更する形になりますね。既存コンポーネントの再利用はできますが、レイアウト構造は大きく変わります。

---

### 2. デザイン方針の議論

#### 2.1 レイアウト構成

**佐藤（UI）**: オリジナルUIの構成を分析すると：

```
┌─────────────────────────────────────────────────────┐
│ ヘッダー（青背景、アプリ名「employees」）           │
├─────────────────┬───────────────────────────────────┤
│                 │                                   │
│  社員リスト     │      社員詳細                     │
│  （約30%幅）    │      （約70%幅）                  │
│                 │                                   │
│  - 写真（丸型） │      2カラムグリッドで情報表示    │
│  - 名前         │      右上に編集・削除ボタン       │
│  - 社員番号     │                                   │
│  - 部署/チーム  │                                   │
│                 │                                   │
└─────────────────┴───────────────────────────────────┘
```

**田中（UX）**: このレイアウトのメリットは：
1. **比較のしやすさ**: 一覧を見ながら詳細を確認できる
2. **操作の連続性**: クリックするだけで右側が切り替わり、スムーズ
3. **コンテキスト維持**: 検索・フィルタ状態を維持したまま詳細確認

**鈴木（FE）**: 懸念点として、**レスポンシブ対応**があります。スマートフォンでは2カラムは厳しい。

**佐藤（UI）**: モバイルでは従来通りページ遷移にフォールバックしましょう。ブレークポイントは `lg:1024px` 以上で2カラム、それ未満は現行のカード＋ページ遷移。

**山田（PM）**: それで行きましょう。デスクトップユーザーが主なので、まずはそこを最適化。

---

#### 2.2 カラースキーム

**佐藤（UI）**: オリジナルはヘッダーが **青系（#0078D4 に近い色）** ですね。現行システムの色と比較すると：

| 要素 | オリジナル | 現行 | 提案 |
|------|-----------|------|------|
| ヘッダー背景 | 青（#0078D4系） | 白/グレー | 青に変更 |
| 選択行 | 青ハイライト | - | 青ハイライト |
| アクセント | 青 | 青 | 維持 |

**田中（UX）**: 青は「信頼性」「プロフェッショナル」の印象を与えます。業務システムには適切ですね。

**鈴木（FE）**: Tailwind CSSで `bg-blue-600` や `bg-blue-700` あたりでしょうか。カスタムカラーを定義してもいいですが。

**佐藤（UI）**: ブランドカラーとして `primary: #0078D4` を定義しましょう。Tailwindの設定に追加します。

---

#### 2.3 社員リストのデザイン

**佐藤（UI）**: リストアイテムの構成：

```
┌─────────────────────────────────────┐
│ [写真]  名前            社員番号    │
│  (丸型) 部署/チーム名         [>]   │
└─────────────────────────────────────┘
```

**田中（UX）**: 写真は **丸型（円形）** がいいですね。親しみやすさと、スペース効率のバランスが良い。サイズは40-48px程度。

**佐藤（UI）**: 選択中の行は左ボーダー（4px、青）で強調。背景も薄い青にして、どれを選択中かを明確に。

**鈴木（FE）**: リストはスクロール可能にして、`overflow-y-auto` で高さを固定。バーチャルスクロールも検討しますか？

**山田（PM）**: 社員数は最大で数百人程度なので、バーチャルスクロールは過剰かも。通常のスクロールで十分でしょう。

---

#### 2.4 社員詳細のレイアウト

**佐藤（UI）**: 詳細画面は **2カラムグリッド** で情報を配置：

```
┌─────────────────────────────────────────────────┐
│ 社員番号（見出し）                    [編集][削除]│
├─────────────────────┬───────────────────────────┤
│ 社員番号            │ 契約形態                  │
│ 200214              │ 正社員                    │
├─────────────────────┼───────────────────────────┤
│ 氏名                │ フリガナ                  │
│ 高田 敬子           │ タカタ ケイコ             │
├─────────────────────┼───────────────────────────┤
│ ...                 │ ...                       │
└─────────────────────┴───────────────────────────┘
```

**田中（UX）**: ラベルと値を縦に並べるのは読みやすいですね。ラベルは小さめ・グレー、値は通常サイズ・黒で視覚的階層を作る。

**佐藤（UI）**: 写真は右下に配置して、主要情報の視認性を優先。サイズは120x160px程度で。

**鈴木（FE）**: CSS Gridで `grid-cols-2 gap-4` のシンプルな構成で実装できます。

---

#### 2.5 フィルター・検索UI

**田中（UX）**: オリジナルUIでは、フィルターがリストの上部に配置されていますね。コンパクトで良い。

**佐藤（UI）**: 提案：
- **1行目**: ステータス選択（在籍/全員など）＋ 検索モード選択（全文検索）
- **2行目**: 検索テキストボックス ＋ 追加ボタン

**鈴木（FE）**: 現行の `EmployeeFilter` コンポーネントをコンパクト化する形ですね。アコーディオンで詳細フィルタを隠すのもアリ。

**山田（PM）**: タグ検索は頻繁に使う機能なので、アクセスしやすくしてほしいという声があります。

**田中（UX）**: では、基本フィルタはコンパクトに、タグ検索は展開式で。使用頻度で段階的に表示。

---

### 3. 追加機能のUI（案件・企業・タグ・ユーザー管理）

**山田（PM）**: 今後Phase 2で追加される機能のUIも考慮しておきたいです。

**佐藤（UI）**: ナビゲーション構造の提案：

```
┌─────────────────────────────────────────────────────┐
│ [ロゴ] employees   [社員][企業][案件][タグ][ユーザー][ログアウト] │
└─────────────────────────────────────────────────────┘
```

**田中（UX）**: タブ形式のナビゲーションがいいですね。現在位置が明確になる。権限によって表示/非表示を制御。

**鈴木（FE）**: 現行のHeaderコンポーネントを拡張。React Routerの `NavLink` でアクティブ状態を管理。

**佐藤（UI）**: 各モジュールも同じマスターディテール構造を採用すれば、UIの一貫性が保たれます。

### 5. 決定事項まとめ

1. **レイアウト**: デスクトップはマスターディテール型（左30%:右70%）、モバイルは従来のページ遷移
2. **カラー**: ヘッダーは青系（#0078D4）、選択行も青ハイライト
3. **社員リスト**: 丸型写真、名前＋社員番号＋部署、選択行は左ボーダー強調
4. **社員詳細**: 2カラムグリッド、ラベル上・値下の縦配置、写真は右下
5. **フィルター**: コンパクト化、タグ検索は展開式
6. **ナビゲーション**: タブ形式で各モジュールへアクセス、権限による表示制御
7. **UI一貫性**: 全モジュールでマスターディテール構造を採用
