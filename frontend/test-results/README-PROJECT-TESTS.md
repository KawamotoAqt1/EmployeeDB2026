## テスト実施の流れ

### Step 1: 準備

1. バックエンドとフロントエンドのサーバーを起動

```bash
# Terminal 1: バックエンド
cd backend
npm run dev

# Terminal 2: フロントエンド
cd frontend
npm run dev
```

2. ブラウザで http://localhost:5173 にアクセス

### Step 2: テスト実施

1. `test-execution-guide.md` を開く
2. セクション1から順番に実施
3. 各チェックボックスにチェックを入れる
4. 問題を発見したら記録する

### Step 3: 結果記録

1. `project-management-tests.md` に結果を記録
   - 成功: ✅
   - 失敗: ❌
   - スキップ: ⏭️

2. 発見した問題を詳細に記録

### Step 4: レポート作成

1. `test-completion-report.md` を更新
2. 承認者の署名を取得
3. リリース判定を記録

## 推奨されるテストシナリオ

### シナリオ1: 基本的なCRUD操作（必須）

1. 案件を新規登録
2. 登録した案件を一覧で確認
3. 案件の詳細を表示
4. 案件情報を編集
5. 案件を削除

**想定時間**: 10分

### シナリオ2: 社員参画管理（必須）

1. 案件詳細を表示
2. 社員を追加（役割、期間、稼働率を設定）
3. 参画社員一覧に表示されることを確認
4. 社員詳細画面で参画履歴を確認
5. 社員を削除

**想定時間**: 10分

### シナリオ3: 検索・フィルタ（推奨）

1. キーワードで検索
2. ステータスでフィルタ
3. 契約形態でフィルタ
4. 複合条件で検索
5. リセットして全件表示

**想定時間**: 10分

### シナリオ4: 権限テスト（必須）

1. ADMIN権限でログイン → すべての操作を確認
2. EDITOR権限でログイン → すべての操作を確認
3. VIEWER権限でログイン → 閲覧のみを確認

**想定時間**: 15分

### シナリオ5: エラーハンドリング（推奨）

1. 必須項目なしで登録を試行
2. 存在しない案件IDにアクセス
3. ネットワークエラーの挙動を確認

**想定時間**: 10分

**合計想定時間**: 約55分（必須のみは35分）
