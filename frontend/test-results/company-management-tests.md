## テスト実施ガイド

### テスト実施前の準備

1. **環境準備**
   ```bash
   # バックエンド起動
   cd backend
   npm run start:dev

   # フロントエンド起動
   cd frontend
   npm run dev
   ```

2. **テストデータ準備**
   - 複数の企業データを登録(20件以上推奨)
   - 各ステータス(ACTIVE, INACTIVE, TERMINATED)のデータを用意
   - 拠点・部署・担当窓口が登録されている企業を用意
   - 複数の権限レベルのユーザーアカウントを用意(ADMIN, EDITOR, VIEWER)

3. **ブラウザ準備**
   - Chrome/Edge/Firefoxの最新版
   - 開発者ツールを開いておく(Network, Consoleタブ)

### テスト実施手順

1. **テストケースごとに実施**
   - 「手順」の内容を正確に実行
   - 「期待結果」と実際の動作を比較
   - 結果を記録(成功/失敗)
   - 失敗の場合は詳細をバグレポートに記載

2. **スクリーンショット取得**
   - 重要な画面は証跡としてスクリーンショットを保存
   - 保存先: `frontend/test-results/screenshots/`

3. **エラー確認**
   - コンソールエラーがないか確認
   - ネットワークエラーがないか確認

4. **結果記録**
   - 各テストケースの「結果」列に成功/失敗を記録
   - 「備考」列に気づいた点を記載

### よくある問題と対処法

| 問題 | 原因 | 対処法 |
|------|------|--------|
| データが表示されない | バックエンドが起動していない | `npm run start:dev` でバックエンドを起動 |
| 401エラー | 認証トークンが無効 | 再ログインする |
| 404エラー | APIエンドポイントが間違っている | バックエンドのログを確認 |
| CORSエラー | フロントエンドとバックエンドのポートが異なる | `.env` ファイルを確認 |
