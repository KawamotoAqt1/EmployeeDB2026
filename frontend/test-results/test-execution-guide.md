# Phase 2 案件管理機能 テスト実施ガイド

## 目的

このガイドは、Phase 2の案件管理機能のテストを実施するための手順書です。

## 前提条件

1. バックエンドサーバーが起動している (http://localhost:3000)
2. フロントエンドサーバーが起動している (http://localhost:5173)
3. データベースに基本的なテストデータが登録されている

## テスト準備

### 1. サーバーの起動

```bash
# Terminal 1: バックエンド
cd backend
npm run dev

# Terminal 2: フロントエンド
cd frontend
npm run dev
```

### 2. テストユーザーの準備

以下の権限でテストを実施します：

| ユーザー | メールアドレス | パスワード | 権限 |
|---|---|---|---|
| Admin | admin@example.com | (設定されているパスワード) | ADMIN |
| Editor | editor@example.com | (設定されているパスワード) | EDITOR |
| Viewer | viewer@example.com | (設定されているパスワード) | VIEWER |

### 3. テストデータの準備

#### 企業データ
1. 部署を持つ企業を1つ以上登録
2. 部署を持たない企業を1つ登録
3. 取引停止ステータスの企業を1つ登録

#### 社員データ
1. 通常の社員を3名以上登録

## テスト実施手順

### セクション1: 案件一覧画面のテスト

#### 1.1 基本表示のテスト

1. ブラウザで http://localhost:5173 にアクセス
2. ログイン画面でEDITOR権限のユーザーでログイン
3. 左メニューから「案件管理」をクリック
4. 以下を確認：
   - [ ] 案件がカード形式で表示される
   - [ ] 案件名が表示される
   - [ ] 企業名が表示される
   - [ ] ステータスバッジが表示される
   - [ ] 契約形態が表示される
   - [ ] 契約期間が表示される
   - [ ] 参画人数が表示される（参画社員がいる場合）
   - [ ] 「X件の案件が登録されています」と表示される
   - [ ] 「新規登録」ボタンが表示される

#### 1.2 検索・フィルタ機能のテスト

1. キーワード検索テスト
   - [ ] 案件名の一部を入力して「検索」をクリック
   - [ ] 該当する案件のみが表示されることを確認
   - [ ] 企業名の一部を入力して「検索」をクリック
   - [ ] 該当する案件のみが表示されることを確認

2. ステータスフィルタテスト
   - [ ] ステータスを「稼働中」に設定して「検索」
   - [ ] 稼働中の案件のみが表示されることを確認
   - [ ] 「すべて」に戻して検索

3. 契約形態フィルタテスト
   - [ ] 契約形態を「派遣」に設定して「検索」
   - [ ] 派遣の案件のみが表示されることを確認

4. リセット機能テスト
   - [ ] フィルタを設定した状態で「リセット」をクリック
   - [ ] すべてのフィルタがクリアされることを確認
   - [ ] 全案件が表示されることを確認

#### 1.3 ページネーション機能のテスト（20件以上の案件がある場合）

1. [ ] 「次へ」ボタンをクリックして次のページに移動
2. [ ] ページ番号が更新されることを確認
3. [ ] ページトップにスクロールされることを確認
4. [ ] 「前へ」ボタンをクリックして前のページに戻る
5. [ ] 1ページ目では「前へ」ボタンが無効化されていることを確認
6. [ ] 最終ページでは「次へ」ボタンが無効化されていることを確認

### セクション2: 案件登録画面のテスト

#### 2.1 新規登録画面への遷移

1. [ ] 案件一覧から「新規登録」ボタンをクリック
2. [ ] 案件登録画面が表示される
3. [ ] タイトルが「案件登録」と表示される

#### 2.2 フォーム表示の確認

1. [ ] 「基本情報」セクションが表示される
2. [ ] 「企業・部署」セクションが表示される
3. [ ] 「契約期間」セクションが表示される
4. [ ] 「備考」セクションが表示される
5. [ ] 案件名、企業、契約形態に赤い「*」が表示される（必須項目）
6. [ ] ステータスのデフォルト値が「稼働中」になっている
7. [ ] 契約形態のデフォルト値が「派遣」になっている

#### 2.3 企業・部署の動的選択テスト

1. [ ] 企業選択欄をクリックして企業を選択
2. [ ] 部署選択欄が表示される（企業に部署が登録されている場合）
3. [ ] 選択した企業の部署のみが選択肢に表示される
4. [ ] 部署を選択
5. [ ] 別の企業を選択
6. [ ] 部署の選択がクリアされることを確認
7. [ ] 新しい企業の部署が選択肢に表示される

#### 2.4 案件の新規登録テスト

1. 以下の情報を入力：
   - 案件名: 「テスト案件A」
   - 案件説明: 「テスト用の案件です」
   - ステータス: 「稼働中」
   - 契約形態: 「SES」
   - 企業: （任意の企業を選択）
   - 部署: （任意の部署を選択、なければスキップ）
   - 開始日: 「2026-01-01」
   - 終了日: 「2026-12-31」
   - 備考: 「テスト登録」

2. [ ] 「登録」ボタンをクリック
3. [ ] ローディング表示が出る
4. [ ] 案件一覧画面に遷移する
5. [ ] 登録した案件が一覧に表示される

#### 2.5 バリデーションテスト

1. [ ] 案件名を空にして「登録」をクリック → エラーが表示される
2. [ ] 企業を未選択で「登録」をクリック → エラーが表示される

### セクション4: 社員参画管理のテスト

#### 4.1 社員追加機能のテスト

1. [ ] 案件詳細画面で「社員を追加」ボタンをクリック
2. [ ] モーダルが表示される
3. [ ] タイトルが「社員を案件に追加」と表示される

4. モーダルの内容を確認：
   - [ ] 社員選択欄が表示される
   - [ ] 社員一覧が「氏名 (社員番号)」形式で表示される
   - [ ] 役割入力欄が表示される
   - [ ] 参画開始日（必須）が表示される
   - [ ] 参画終了日が表示される
   - [ ] 稼働率が表示される（デフォルト100）
   - [ ] 備考欄が表示される

5. 社員を追加：
   - [ ] 社員を選択
   - [ ] 役割: 「プロジェクトリーダー」
   - [ ] 参画開始日: 「2026-02-01」
   - [ ] 参画終了日: 「2026-12-31」
   - [ ] 稼働率: 「80」
   - [ ] 備考: 「テスト参画」
   - [ ] 「追加する」ボタンをクリック

6. [ ] ローディング表示が出る
7. [ ] モーダルが閉じる
8. [ ] 参画社員一覧に追加した社員が表示される
9. [ ] 社員名、役割、参画期間、稼働率が正しく表示される

#### 4.2 社員追加のバリデーションテスト

1. [ ] 社員を選択せずに追加しようとする → 「追加する」ボタンが無効化されている
2. [ ] 参画開始日を入力せずに追加しようとする → 「追加する」ボタンが無効化されている
3. [ ] 稼働率に101を入力 → HTML5バリデーションで拒否される
4. [ ] 稼働率に-1を入力 → HTML5バリデーションで拒否される

#### 4.3 複数の社員を追加

1. [ ] 「社員を追加」ボタンを再度クリック
2. [ ] 別の社員を選択
3. [ ] 参画開始日のみ入力（役割、終了日、備考は空）
4. [ ] 「追加する」ボタンをクリック
5. [ ] 2人目の社員が追加される
6. [ ] 稼働率がデフォルトの100%で表示される

#### 4.4 社員削除機能のテスト

1. [ ] 参画社員の削除ボタン（ゴミ箱アイコン）をクリック
2. [ ] 確認ダイアログが表示される
3. [ ] メッセージが「この参画情報を削除してもよろしいですか？」と表示される
4. [ ] 「キャンセル」をクリック → 削除されない
5. [ ] 再度削除ボタンをクリック
6. [ ] 「OK」をクリック
7. [ ] 参画社員が一覧から削除される
