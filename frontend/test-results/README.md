## テスト実施の流れ

### ステップ1: 環境準備

1. **バックエンド起動**
   ```bash
   cd C:\dev\EmployeeDB\backend
   npm run start:dev
   ```

2. **フロントエンド起動**
   ```bash
   cd C:\dev\EmployeeDB\frontend
   npm run dev
   ```

3. **テストデータ作成**
   ```bash
   psql -U postgres -d employeedb -f test-data-setup.sql
   ```

### ステップ2: テスト実施

**初回テスト / 完全テスト**
- `company-management-tests.md` を使用
- すべてのテストケースを実施
- 実施時間: 約2-3時間

**クイックテスト / 日常的な確認**
- `quick-test-guide.md` を使用
- 主要機能のみを確認
- 実施時間: 約40-50分

### ステップ3: 結果記録

- `test-execution-log.md` に結果を記録
- 発見したバグをバグレポートに記載
- スクリーンショットを `screenshots/` に保存

### ステップ4: 後処理

**テストデータ削除**
```sql
DELETE FROM contacts WHERE id LIKE 'test-contact-%';
DELETE FROM departments WHERE id LIKE 'test-dept-%';
DELETE FROM offices WHERE id LIKE 'test-office-%';
DELETE FROM companies WHERE id LIKE 'test-company-%';
```

## よくある質問

### Q: テストデータが表示されない

**A**: 以下を確認してください:
1. バックエンドが起動しているか
2. テストデータ作成スクリプトが実行されたか
3. データベース接続設定が正しいか

### Q: 権限テストができない

**A**: 複数の権限レベルのユーザーアカウントが必要です:
- ADMIN権限のユーザー
- EDITOR権限のユーザー
- VIEWER権限のユーザー

これらは事前にSwagger UIまたは直接DBに登録してください。

### Q: ページネーションテストができない

**A**: 21件以上のデータが必要です。`test-data-setup.sql` では24件の企業データを作成します。

### Q: テストにどのくらい時間がかかる?

**A**:
- 完全テスト: 2-3時間
- クイックテスト: 40-50分
- 環境準備: 5-10分

### Q: 自動テストはないのか?

**A**: 現在は手動テストのみです。将来的には以下の自動テスト追加を検討:
- E2Eテスト(Playwright, Cypress)
- 統合テスト(Jest, React Testing Library)
- ビジュアルリグレッションテスト

## バグ報告フォーマット

バグを発見した場合は、以下の情報を記録してください:

```markdown
## バグ #X

- **発見日**: YYYY-MM-DD
- **テストID**: T1-2-3
- **重要度**: 致命的 / 重大 / 中程度 / 軽微
- **再現手順**:
  1. ステップ1
  2. ステップ2
  3. ステップ3
- **期待動作**: 期待される動作
- **実際の動作**: 実際に起きた動作
- **スクリーンショット**: screenshots/bug-X.png
- **コンソールエラー**: エラーメッセージ
- **環境**: OS, ブラウザ
- **ステータス**: 未対応 / 対応中 / 修正完了 / 確認済み
```

## 次のステップ

テストが完了したら:

1. **結果レビュー**: テスト結果を確認
2. **バグ修正**: 発見されたバグを修正
3. **再テスト**: 修正後に該当テストケースを再実施
4. **ドキュメント更新**: 必要に応じてテスト設計書を更新
5. **Phase 3準備**: 案件管理機能のテスト設計を開始
