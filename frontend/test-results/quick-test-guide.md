# クイックテストガイド - 企業管理機能

このガイドは、企業管理機能を短時間で動作確認するための簡易版テスト手順です。

---

## 環境準備 (5分)

### 1. バックエンド起動

```bash
cd C:\dev\EmployeeDB\backend
npm run start:dev
```

確認: http://localhost:3000/api で Swagger UI が表示される

### 2. フロントエンド起動

```bash
cd C:\dev\EmployeeDB\frontend
npm run dev
```

確認: http://localhost:5173 でログイン画面が表示される

### 3. ログイン

- **ADMIN権限**: 全機能テスト用
- **VIEWER権限**: 権限制限テスト用

---

## 基本機能テスト (15分)

### STEP 1: 企業一覧表示

**URL**: http://localhost:5173/companies

**確認項目**:
- [ ] 企業一覧が表示される
- [ ] 企業名、業種、ステータス、登録日が見える
- [ ] ステータスバッジの色が正しい(ACTIVE=青、INACTIVE=黄、TERMINATED=灰)
- [ ] 登録件数が表示される
- [ ] 「新規登録」ボタンが表示される(EDITOR/ADMIN権限)

### STEP 2: 検索機能

**手順**:
1. 企業名フィールドに任意のキーワードを入力
2. 「検索」ボタンをクリック

**確認項目**:
- [ ] 該当する企業のみが表示される
- [ ] 検索結果件数が更新される

**手順**:
3. 「リセット」ボタンをクリック

**確認項目**:
- [ ] 検索条件がクリアされる
- [ ] 全件表示に戻る

### STEP 3: 企業詳細表示

**手順**:
1. 任意の企業の「詳細」リンクをクリック

**確認項目**:
- [ ] 企業の基本情報が表示される(企業名、業種、ステータス、Webサイト、備考)
- [ ] 拠点一覧が表示される(拠点がある場合)
- [ ] 部署階層がツリー構造で表示される(部署がある場合)
- [ ] 担当窓口一覧が表示される(担当者がいる場合)
- [ ] 関連案件一覧が表示される(案件がある場合)
- [ ] 「編集」「削除」ボタンが表示される(EDITOR/ADMIN権限)

### STEP 4: 企業新規登録

**手順**:
1. 企業一覧で「新規登録」ボタンをクリック
2. 以下の情報を入力:
   - 企業名: 「テスト株式会社」
   - 企業名カナ: 「テストカブシキガイシャ」
   - 業種: 「IT・ソフトウェア」
   - ステータス: 「取引中」
   - Webサイト: 「https://example.com」
   - 備考: 「テスト用企業」
3. 「登録」ボタンをクリック

**確認項目**:
- [ ] エラーなく登録できる
- [ ] 企業一覧画面に遷移する
- [ ] 登録した企業が一覧に表示される

### STEP 5: 拠点追加

**手順**:
1. 新規登録画面または編集画面を開く
2. 拠点情報セクションの「追加」ボタンをクリック
3. 以下の情報を入力:
   - 拠点名: 「本社」
   - 郵便番号: 「100-0001」
   - 住所: 「東京都千代田区千代田1-1」
   - 電話番号: 「03-1234-5678」
   - 主要拠点: チェック
4. 「登録」または「更新」ボタンをクリック

**確認項目**:
- [ ] 拠点が正しく登録される
- [ ] 詳細画面で拠点情報が表示される
- [ ] 主要拠点バッジが表示される

### STEP 6: 担当窓口追加

**手順**:
1. 新規登録画面または編集画面を開く
2. 担当窓口セクションの「追加」ボタンをクリック
3. 以下の情報を入力:
   - 氏名: 「山田太郎」
   - 氏名カナ: 「ヤマダタロウ」
   - 役職: 「部長」
   - メールアドレス: 「yamada@example.com」
   - 電話番号: 「03-1234-5678」
   - 携帯番号: 「090-1234-5678」
   - 主担当: チェック
4. 「登録」または「更新」ボタンをクリック

**確認項目**:
- [ ] 担当窓口が正しく登録される
- [ ] 詳細画面で担当窓口情報が表示される
- [ ] 主担当バッジが表示される
- [ ] メールアドレスがmailtoリンクになっている

### STEP 7: 企業編集

**手順**:
1. 企業詳細画面で「編集」ボタンをクリック
2. 企業名を変更する(例: 「テスト株式会社」→「テスト株式会社 (更新)」)
3. 「更新」ボタンをクリック

**確認項目**:
- [ ] エラーなく更新できる
- [ ] 企業一覧画面に遷移する
- [ ] 変更内容が反映されている

### STEP 8: 企業削除

**手順**:
1. 削除したい企業の詳細画面を開く
2. 「削除」ボタンをクリック
3. 削除確認モーダルが表示される
4. 企業名を確認する
5. 「削除する」ボタンをクリック

**確認項目**:
- [ ] 削除確認モーダルが表示される
- [ ] 警告文が表示される
- [ ] 削除が実行される
- [ ] 企業一覧画面に遷移する
- [ ] 削除した企業が一覧から消えている

### STEP 9: ページネーション

**前提**: 21件以上のデータがある状態

**手順**:
1. 企業一覧画面を表示
2. ページネーションの「次へ」ボタンまたはページ番号をクリック

**確認項目**:
- [ ] 次のページが表示される
- [ ] ページ情報(「全X件中Y-Z件を表示」)が正しい
- [ ] ページトップにスクロールされる
- [ ] 最初のページで「前へ」ボタンが無効
- [ ] 最後のページで「次へ」ボタンが無効

## トラブルシューティング

### 問題: 企業一覧が空

**原因**: データが登録されていない
**対処**: Swagger UIまたは画面から企業を登録する

### 問題: 「新規登録」ボタンが表示されない

**原因**: VIEWER権限でログインしている
**対処**: EDITOR/ADMIN権限でログインし直す

### 問題: API接続エラーが表示される

**原因**: バックエンドが起動していない
**対処**: `cd backend && npm run start:dev` で起動する

### 問題: ページが真っ白

**原因**: JavaScriptエラーが発生している
**対処**:
1. ブラウザのコンソールを確認
2. ブラウザのキャッシュをクリア
3. フロントエンドを再起動

### 問題: データが保存されない

**原因**: データベース接続エラー
**対処**:
1. PostgreSQLが起動しているか確認
2. `.env` ファイルの `DATABASE_URL` を確認
3. バックエンドのログを確認
